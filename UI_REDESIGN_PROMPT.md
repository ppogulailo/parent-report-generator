## 4. Hard constraints ŌĆö DO NOT break (the app crashes or malfunctions otherwise)

These are non-negotiable and reflect real bugs already fixed:

1. **Browser-translation gate.** The results/plan region **must** keep `translate="no"`
   permanently. The plan is generated by the backend in the language the user selected; it
   must never be re-translated by Chrome/Google Translate. (Re-translation previously raced
   React's streaming DOM updates, dropped sentence tails, and crashed the page.)
2. **No bare text nodes inside interactive/dynamic controls.** Any label that changes
   (the Generate button's label, the progress label) must be wrapped in its own element
   (e.g. `<span>`), never a bare text child sitting next to another element. (Browser
   translation relocates bare text into `<font>` wrappers, which makes React throw
   "insertBefore ŌĆ” not a child.")
3. **Preserve the streaming/SSE contract.** Results come from `POST /api/report/stream` as
   Server-Sent Events with event names: `scores`, `text`, `reset`, `error`, `done`. Sections
   are parsed out of the accumulated text by locating section-marker strings. On `reset` or
   `error`, the partial plan must be **discarded** (never shown as if final). Keep this logic.
4. **Language via routing.** Keep `/en` and `/es` as the source of truth; the switcher
   navigates between them and is disabled during loading.
5. **Accessibility must be preserved or improved.** Radio-group semantics for the language
   switcher and the 1ŌĆō4 scales; `aria-live` on the streaming status; `aria-busy` on the
   submit button; real `<label>`s tied to inputs; visible focus states; honor
   `prefers-reduced-motion` (the typing cursor / spinners must have a reduced-motion path).
6. **Stay a Next.js 15 / React 19 client component.** Keep the same component prop
   (`language`) and keep consuming the existing i18n exports (`STRINGS`, `QUESTIONS`,
   `ANSWER_LABELS`, `SECTION_LABELS_BY_LANG`, `SECTION_MARKERS_BY_LANG`, `domainLabel`).
   Don't change the request payload shape: `{ responses: number[24], language, crisis? }`.
